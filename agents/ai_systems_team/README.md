# AI Systems Team

The **AI Systems Team** is a sub-team of the software engineering organization focused on one job:

> Take a product/problem spec as input and produce a goal-aligned AI agent system implementation plan (and build package) that can execute the requested outcomes.

This team is designed as a reusable **agent factory workflow**. Instead of building one-off assistants, it repeatedly turns clear specs into production-ready agent systems.

## What this team does

Given a spec, the team will:

1. Clarify goals, constraints, and success criteria.
2. Decompose the desired behavior into agent roles.
3. Design orchestration strategy (sequential, parallel, hierarchical, or event-driven).
4. Define tool contracts and external integrations.
5. Establish memory/state strategy (session, long-term, retrieval, and audit trail).
6. Add safety, governance, and human-approval checkpoints.
7. Produce implementation artifacts (prompts, contracts, test scenarios, observability plan).

## Core sub-agents

- **Spec Intake Agent**
  - Parses the user-provided spec.
  - Extracts goals, non-goals, assumptions, and constraints.
- **Solution Architect Agent**
  - Selects the agent topology and workflow pattern.
  - Defines handoffs and escalation paths.
- **Capabilities Planner Agent**
  - Maps each requirement to tools, skills, memory, and models.
- **Evaluation Agent**
  - Creates acceptance tests, adversarial tests, and measurable KPIs.
- **Safety & Governance Agent**
  - Applies guardrails, policy checks, and human-gate requirements.
- **Build Coordinator Agent**
  - Packages the final output into an implementation-ready bundle.

## Inputs

A spec should include:

- Problem statement
- Desired outcomes
- Must-have constraints (budget, latency, privacy, compliance)
- Allowed and disallowed actions
- Human-in-the-loop requirements
- Quality expectations (accuracy, throughput, reliability)

See `spec_template.md` for a ready-to-use template.

## Outputs

The team outputs a complete **Agent System Blueprint**:

- Agent roster and role definitions
- Orchestration graph and control flow
- Tooling/integration contracts
- State & memory policy
- Safety and approval checkpoints
- Evaluation harness and pass/fail criteria
- Rollout plan (pilot -> staged -> production)

## Recommended delivery phases

1. **Phase 1: Blueprint**
   - Generate architecture and execution design.
2. **Phase 2: Prototype**
   - Implement smallest viable multi-agent loop.
3. **Phase 3: Hardening**
   - Add test coverage, observability, retries, and policy gates.
4. **Phase 4: Productionization**
   - Add deployment automation and operations runbooks.

## Definition of done

An AI system generated by this team is done when:

- Every requested goal maps to one or more executable agent behaviors.
- Safety constraints are enforced at runtime.
- Human approval points exist where risk is non-trivial.
- Acceptance tests pass against the original spec.
- Metrics and logs are sufficient for post-deployment debugging.

## Example mission statement

"Build and operate a specialized agent team that consumes structured specs and outputs production-ready AI agent systems that reliably achieve the specified goals under explicit safety, quality, and operational constraints."
